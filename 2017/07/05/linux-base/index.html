文件与目录管理

ls

stat 显示指定文件的详细信息，比ls更详细

pwd 获取当前目录所在的路径

mkdir -p 创建目录，若无父目录，则创建p(parent)

树形结构列出指定目录下的所有文件，包括子目录里的文件

cat 显示全部内容，结合grep可以查找内容

more 分页显示文本文件内容
less 支持上下滑动，左右移动显示内容

显示文件头、尾内容

一般模式：
上下翻页：Ctrl+f、Ctrl+b、
跳到当前行最后字符处：Fn+右键
跳到当前行最前面字符处：Fn+左键、0
查找字符images：/image n继续查找
删除行：dd ndd（n代表删除或复制的行数）
复制行：yy nyy
粘贴行：p、P
复原前一个操作：u
重做上一个操作：ctrl+r
显示行号：:set nu
字符全局替换：:%s/foo/bar/g :{作用范围}s/{目标}/{替换}/{替换标志}

从远程拷贝目录到本地
scp -r root@10.10.10.10:/opt/soft/mongodb /opt/soft/
上传本地目录到远程机器指定目录
scp -r /opt/soft/mongodb root@10.10.10.10:/opt/soft/scptest

chmod 修改一个文件的权限
chown 修改一个文件所归属的用户或组

进程和网络管理

ps

top

netstat

ss

系统管理命令

du

watch

who

whoami

uname

rpm

alias showmeit=”ps -aux” 命令重命名
unaliax showmeit 解除使用

nohup command > myout.file 2>&1 &
stderr和stdout被重定向到myout.file文件中
使用Ctrl + C发送SIGINT信号，程序关闭；
关闭session发送SIGHUP信号，程序免疫；
平时经常使用nohup和&配合来启动程序。双同免疫

script -q myfile 静默模式记录，exit 退出记录

crontab文件的含义：用户所建立的crontab文件中，每一行都代表一项任务，每行的每个字段代表一项设置，它的格式共分为六个字段，前五段是时间设定段，第六段是要执行的命令段，格式如下：minute hour day month week command

每五分钟执行  */5 * * * *

每小时执行  0 * * * *

每天执行    0 0 * * *

每周六、周日的1:10重启smb *10 1 * * 6,0 /etc/init.d/smb restart*

创建cron脚本用法

  • 第一步，vi创建cron脚本,命名为crontest.cron，写入命令0 9 echo “Good morning.” >> /tmp/test.txt* 每天早上9点执行打印一次命令
  • 第二步：添加定时任务。执行命令 crontab crontest.cron 搞定

  • 第三步：crontab -l 查看定时任务是否成功或者检测/var/spool/cron下是否生成对应cron脚本

    其它用法

  • crontab -r 删除该用户的定时任务

  • crontab -e 编辑该用户的定时任务

下载并以不同的文件名保存
wget -O harbor.tgz https://storage.googleapis.com/harbor-releases/release-1.7.0/harbor-offline-installer-v1.7.1.tgz
后台下载
wget -b url
